# 冻存盒管理系统本地版使用说明

本地版用于在实验室电脑上直接运行，不依赖 Vercel，不连接 Supabase，不需要登录。数据只保存在当前电脑的当前浏览器中。

## 首次准备

1. 安装 Node.js LTS：<https://nodejs.org/>
2. 解压或下载本项目文件夹。
3. 双击启动脚本会自动安装依赖。也可以提前在项目文件夹中手动安装：

```bash
npm install --registry=https://registry.npmmirror.com
```

如果电脑已经安装 pnpm，也可以使用：

```bash
pnpm install
```

## macOS 打开方式

双击：

```text
打开冻存盒管理系统.command
```

如果 macOS 提示没有权限，在项目文件夹中运行一次：

```bash
chmod +x 打开冻存盒管理系统.command
```

然后再双击打开。

首次运行会自动安装依赖，可能需要几分钟。

## Windows 打开方式

双击：

```text
打开冻存盒管理系统.bat
```

系统会自动打开浏览器访问：

```text
http://localhost:3000
```

首次运行会自动安装依赖，可能需要几分钟。

## 数据保存和备份

- 数据保存在当前浏览器的本地存储中。
- 不同电脑、不同浏览器之间不会自动同步。
- 清理浏览器数据、换浏览器、重装系统，都可能导致数据丢失。
- 建议定期在系统设置中导出“完整 JSON”作为备份。
- 需要换电脑时，先在旧电脑导出 JSON，再在新电脑导入。

## 常见问题

### 打开后仍然是登录页

请确认是通过 `打开冻存盒管理系统.command` 或 `打开冻存盒管理系统.bat` 启动，而不是直接运行普通线上版命令。

### 提示没有 Node.js

安装 Node.js LTS 后重新打开。

### 提示没有 node_modules

新版启动脚本会自动安装依赖。如果自动安装失败，请在项目文件夹运行：

```bash
npm install --registry=https://registry.npmmirror.com
```

### 端口 3000 被占用

如果系统已经打开，浏览器访问 `http://localhost:3000` 即可。否则关闭占用端口的程序后重新双击启动脚本。

### 数据不见了

优先检查是否换了浏览器或清理过浏览器数据。本地版没有云端恢复能力，只能从之前导出的完整 JSON 备份恢复。
