# 冻存盒管理系统

适配 Mac 与 iPhone 的中文 PWA，支持冰箱—层架—冻存盒—孔位管理、多孔位同时登记、样品搜索、出库、软删除、审计记录、CSV 导入和完整导出。登记信息包括样品名称、样品类型、颜色、时间和复苏皿规格。

## 本地运行

在 Finder 中双击 `一键打开冻存盒.command`，系统会自动启动并打开浏览器。重复双击不会重复启动服务。

也可在终端手动运行：

```bash
pnpm install
pnpm dev
```

侧栏提供“登录云端”入口。未配置环境变量时仍可使用本地演示模式，数据保存在当前浏览器的 `localStorage`；登录入口会提示云端尚未配置。该模式仅用于验收，不应作为正式科研数据存储。

## 配置 Supabase

1. 创建 Supabase 项目，在 SQL Editor 运行 `supabase/migrations/0001_initial.sql`。
2. 复制 `.env.example` 为 `.env.local`，填写项目 URL 和匿名密钥。
3. 先在 Supabase Authentication 后台创建授权用户。前端 OTP 请求已设置 `shouldCreateUser: false`，未知邮箱不会自动注册。邮箱 OTP 登录仍需按项目邮件设置配置。
4. 重启应用并完成新增、移动、出库、恢复和导出测试。

每个授权邮箱是独立账号，数据库行级权限会隔离不同账号的数据。当前版本不是多人共享同一库存；如需多人共同编辑同一套冻存盒，必须增加团队和成员权限模型。

## 部署到 Vercel

导入本目录，在 Vercel 项目环境变量中添加 `NEXT_PUBLIC_SUPABASE_URL` 和 `NEXT_PUBLIC_SUPABASE_ANON_KEY`。将正式域名加入 Supabase Authentication 的 Redirect URLs。

## 备份要求

- 应用内“完整 JSON”用于人工备份和迁移，不替代数据库备份。
- 在 Supabase 控制台启用与套餐匹配的每日备份；若套餐不支持 30 天保留，使用受控的定时任务执行数据库备份并保存至私有、加密存储。
- 正式录入前必须实际演练一次恢复，确认样本、位置和审计记录均可恢复。

## 数据边界

系统未按临床信息系统标准设计。不要录入患者姓名、住院号、手机号等直接身份信息。若未来处理可识别临床数据，需另行完成伦理、权限分级、加密、访问审计和合规评估。
